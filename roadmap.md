# Roadmap

## Deferred: Direct AI Chatbot Integration

Rather than exporting a prompt file to paste into Claude/ChatGPT, the goal is to embed a chatbot directly in the mock station view so students can practice without leaving the app. The chatbot would use the same three-phase structure as the exported prompt: patient roleplay → examiner roleplay → structured feedback.

The app is a pure client-side SPA (Vite/React, no backend), so all options below involve the user providing credentials that are stored in `localStorage`.

### Options

| Option | Cost | Setup required | Response quality |
|--------|------|----------------|-----------------|
| **Anthropic API key** | API credits (paid) | User pastes key from console.anthropic.com | Best — full Claude models |
| **Google Gemini free tier** | Free | User gets a key from aistudio.google.com (Google account required) | Good — Gemini 1.5 Flash |
| **OpenAI API key** | API credits (paid) | User pastes key from platform.openai.com | Good — GPT-4o |
| **WebLLM (local browser model)** | Free | None — downloads model on first use (~2–4 GB) | Lower — Llama/Phi models |

### Implementation notes for when this is built

- The system prompt is already generated by `generatePrompt()` in `MockExamView.tsx` — reuse this.
- Chat UI: a 5th tab in `MockExamView` or a slide-out panel over the existing tabs.
- API call wiring:
  - Anthropic: `@anthropic-ai/sdk` with `dangerouslyAllowBrowser: true`
  - Gemini: `@google/generative-ai` — supports browser calls natively
  - OpenAI: `openai` SDK with `dangerouslyAllowBrowser: true`
  - WebLLM: `@mlc-ai/web-llm`
- A settings modal (gear icon in the header) where the user enters and saves their key.
- Streaming responses for a better UX (`stream: true` on all major SDKs).
